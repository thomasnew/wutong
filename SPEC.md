## 4. 需求定义
### 4.1 功能需求（Functional Requirements）
- FR-001：过将指定目录中的照片以精美排版的方式展示到页面上;
- FR-002：页面通过类似文件夹菜单栏的形式体现目录中的子目录内容;
- FR-003：每张展示的照片,需要对应保存一些文字和其他数据,例如支持用户点击"喜欢",也可以添加评论;
- FR-004: 支持多用户;
- FR-005: 用户分角色管理: admin 管理员角色,可以操作所有,包括照片和评论,管理用户等; 普通用户角色只能浏览照片,添加评论和点击"喜欢"某张照片;
- FR-006: 前端框架采用 vue3 + vite, 后端可采用 python; 
- FR-007: 数据库采用文件存储就行,例如 json ; 后续数据量大了之后,可方便转为 db,例如 mysql等;
- FR-008: 系统能自动分析照片的创建时间,以及地点等其他信息; 这些信息也存入 json (admin可编辑);
