# 微信小程序规格文档（草案）

## 1. 文档目的

本文档用于定义“梧桐拾光”微信小程序前端方案，在复用现有后端能力的前提下，明确：

- 功能范围
- 技术方案
- 接口约定
- 权限模型
- 数据模型变更
- 非功能需求
- 迭代计划

本文档定稿后再进入开发实施阶段。

---

## 2. 背景与目标

### 2.1 现状

- 现有系统为 `Vue3 + FastAPI` 的 H5/PC 站点。
- 后端已切换到 MySQL，支持用户、照片/视频、点赞、评论、排行榜、后台管理等能力。
- 当前登录主要为本地账号密码登录（token 鉴权）。

### 2.2 小程序目标

- 新增微信小程序前端，代码与现有 `frontend/` 分离，独立目录维护。
- 登录采用微信用户体系。
- 保留现有用户管理能力（admin/user），并新增微信授权准入能力（白名单）。
- 评论和点赞展示微信昵称与微信头像。

### 2.3 非目标（当前阶段不做）

- 不替换现有 H5 前端（并行维护）。
- 不在本阶段重构后端核心业务流程（尽量复用现有接口逻辑）。
- 不在本阶段做复杂推荐算法与消息推送。

---

## 3. 总体范围

### 3.1 范围内

- 小程序前端工程新增（建议目录：`miniapp/`）。
- 微信登录接入（后端对接微信 `code2Session`）。
- 微信白名单准入管理（仅管理员可操作）。
- 榜单/照片墙/详情/点赞/评论能力在小程序落地。
- 管理功能保留（用户管理 + 微信授权管理）。

### 3.2 范围外（可后续）

- 多租户
- 消息订阅推送
- AI 标签识别

---

## 4. 术语与角色

- **local 用户**：系统内原账号密码用户。
- **wechat 用户**：通过微信身份登录的用户。
- **admin**：管理员，具备用户与授权管理能力。
- **user**：普通用户，具备浏览、点赞、评论能力。

---

## 5. 功能需求

## 5.1 登录与鉴权

### 5.1.1 微信登录

- 小程序发起：
  - `wx.login()` 获取 `code`
  - 用户授权获取昵称与头像
- 后端新增微信登录接口，校验微信身份并签发现有 token。

### 5.1.2 白名单准入

- 仅在白名单中的微信用户允许登录。
- 非白名单用户返回明确提示：未获授权，请联系管理员。

### 5.1.3 Token 机制

- 延用后端现有 Bearer Token 鉴权机制。
- 小程序本地安全存储 token（`wx.setStorageSync`）。
- token 失效后自动跳转登录页。

## 5.2 首页（排行榜）

- 展示三类榜单：
  - 最多点赞
  - 最多评论
  - 最近更新
- 支持“更多”查看单榜单完整列表。
- 列表项支持点击进入详情页。

## 5.3 照片墙

- 按目录筛选媒体。
- 支持图片与视频展示。
- 保持现有核心交互：点击进入详情页。

## 5.4 详情页

- 展示：
  - 媒体内容（图片/视频）
  - 标题、描述、地点、拍摄时间等元数据
  - 点赞状态与点赞人数
  - 评论列表
- 支持：
  - 点赞/取消点赞
  - 发表评论
  - 删除本人评论（管理员可删任意评论）

## 5.5 用户与授权管理（管理员）

- 保留现有用户管理功能：
  - 创建用户
  - 重置用户名/密码
  - 删除用户（保持“不能删除最后一个管理员”规则）
- 新增微信授权管理：
  - 新增白名单
  - 启用/禁用授权
  - 删除授权
  - 设定角色（admin/user）

## 5.6 昵称与头像展示

- 点赞列表、评论列表优先展示微信昵称与头像。
- 若无微信资料，则回退系统用户名与默认头像。

## 7. 后端改造需求

## 7.1 数据模型变更（建议）

### users（扩展字段）

- `auth_provider`：`local | wechat`
- `wechat_openid`（唯一，nullable）
- `wechat_unionid`（nullable）
- `wechat_nickname`
- `wechat_avatar_url`

### 新表 wechat_authorizations

- `id`
- `wechat_openid`（唯一）
- `wechat_unionid`（可选）
- `display_name`（管理员备注）
- `role`（admin/user）
- `enabled`（bool）
- `created_at`
- `updated_at`
- `last_login_at`（可选）

## 7.2 新增接口（建议）

- `POST /api/auth/wechat/login`
- `GET /api/admin/wechat-authorizations`
- `POST /api/admin/wechat-authorizations`
- `PATCH /api/admin/wechat-authorizations/{id}`
- `DELETE /api/admin/wechat-authorizations/{id}`

## 7.3 现有接口响应增强

涉及点赞用户、评论用户的返回结构增加：

- `avatar_url`
- `display_name`（微信昵称展示字段）
- `username`（兼容字段）

展示优先级约定（前后端统一）：

- `display_name > username`

---

## 8. 接口契约（草案）

## 8.1 微信登录

`POST /api/auth/wechat/login`

请求体：

```json
{
  "code": "wx-login-code",
  "nickname": "微信昵称",
  "avatar_url": "https://..."
}
```

成功响应：

```json
{
  "token": "xxxxx",
  "user": {
    "id": "uuid",
    "username": "展示名",
    "role": "user",
    "avatar_url": "https://...",
    "auth_provider": "wechat"
  }
}
```

失败响应（示例）：

- `401`：微信身份校验失败
- `403`：未授权登录

## 8.2 点赞/评论用户展示字段（示例）

```json
{
  "liked_users": [
    {
      "user_id": "uuid",
      "username": "微信昵称",
      "avatar_url": "https://..."
    }
  ],
  "comments": [
    {
      "id": "uuid",
      "user_id": "uuid",
      "username": "微信昵称",
      "avatar_url": "https://...",
      "content": "评论内容"
    }
  ]
}
```

## 9. 安全与合规

- 小程序端不得存储明文敏感信息。
- token 存储在小程序本地缓存，登出及时清除。
- 微信昵称、头像属于用户个人信息，需在隐私政策中声明用途。
- 白名单未授权用户不得进入系统业务页面。

## 10. 部署与配置

- 小程序请求域名需配置到微信后台“合法域名”。
- 后端必须提供 HTTPS 可访问域名。
- 若使用微信登录，需配置：
  - `WECHAT_APP_ID`
  - `WECHAT_APP_SECRET`
- 我的计划是在家庭宽带中 ddns 一个 ipv6 地址的域名做后端,前端运行在微信上;域名我会申请好;
- 暂时没有 ipv4 地址,先按照 ipv6 方式;
- Let's Encrypt 用DNS-01 方式证书认证;
- 443端口我需要实际验证,暂留置此问题;

部署前置条件（阶段 A Gate）：

- `wutong.online:443` 外网可达（可完成小程序真实网络访问）
- DNS-01 证书签发成功且可自动续期
- 小程序后台合法域名配置成功（request/upload/download/socket）

IPv6-only 风险与兜底方案：

- 若真机在部分网络环境下无法访问 IPv6 服务，需提供 IPv4 兜底访问链路（云主机反向代理/隧道/CDN 任一可行方案）
- 未完成兜底前，小程序上线范围仅限已验证可达网络
